Flight Service

Individual class project

A CLI application that allows a user to search, book, pay, and cancel flights concurrently.

Java, JUnit, Integration tests, JDBC, MSSQL, Azure

Role Software Engineer
Date August 2020
Link Private repo available upon request

Goals

  • To design and develop a database application that handles concurrent transactions.
  • To minimize security risks when handling client input.

What I Did

Firstly, I designed the Flights database to allow for user creation and login, as well as to keep a persistent log of reservations. Then, using JDBC, I connected the Java application to the MSSQL database hosted on Azure. In addition, I ensured all queries are wrapped in PreparedStatements with parameterized user arguments. This prevents users from breaking the database using SQL commands as input.

Secondly, I scaled up the application by implementing transactions to allow for multiple users on different terminals to use my application concurrently. I ensured that in the case of a deadlock, I re-try the transaction that was blocked.

Lastly, I wrote additional test scripts to ensure correctness and reliability of my application. The tests I wrote include both sequential cases (no conflicting transactions) and concurrent casses.
