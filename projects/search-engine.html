Search Engine

Individual class project

A multi-threaded web-based search engine that processes multi-word queries and static file queries.

C, C++, Make, Google Test

Role Software Engineer
Date January 2020 — March 2020
Link Private repo available upon request

Goals

  • To build and use generic data structures (doubly-linked list and chaining hash table) that can store any arbitrary data type.
  • To parse text files and directories by using provided OS services (specifically Posix).
  • To add concurrency to the search engine.

What I Did

Firstly, I built a doubly-linked list and a chaining hash table. By storing void pointers, these data structures allow for any type of data to be stored. To follow C module conventions, I provided header guards and full documentation in the .h files and hid implementation details in the .c files.

Using those data structures, I built a file system crawler that crawls through a provided directory and folds the contents of each file into an index. The details of this step are difficult to follow, so I will skip them. Essentially, I correctly used and managed multiple nested data structures to build the crawler. I also explicitly allocated and freed memory during the process.

In addition, I improved the engine's performance by:

  • translating in-memory data structures into disk-based, architecture-neutral index file,
  • adding threads to the engine.
