Distributed Database

Pair class project

A distributed, fault-tolerant, sharded key-value database.

Java, JUnit, Lombok, Apache Commons Lang, Google Guava

Role Software Engineer + Chief Debugger
Date March 2020 — June 2020
Link Private repo available upon request

Goals

In this project similar to real-life implementations of distributed systems such as Amazon's DynamoDB or Google's Spanner, I aim to build a distributed database that withstands server failures and network complications such as delayed or dropped messages.

What I Did

I built a fault-tolerant distributed database using various methods such as Primary + Backup and Distributed Concensus. I also balanced workload among servers via sharding, increasing performance and preventing server crashes due to high-traffic. In addition, I implemented dynamic load balancing when servers join and leave, or when shards are moved.

In the Distributed Concensus method, I implemented the multi-instance Paxos algorithm in which all servers replicate each command in the exact order as prompted by the leader. This way, I guarantee high-availability and linearizability.

In addition, I took out countless obscure bugs due to various network complications via backtracing, rubber duck debugging, and good ol' print statements.

What I Learned

This project enhanced my passion for problem-solving given that it is not only difficult, but also extremely obscure to debug. Owning up to the challenge and being able to explain the bugs gave me a great sense of accomplishment and thrill!

I also worked on my pair-programming skills in this project. In working with a project partner, I was able to work on articulating my train of thought and reinforcing my own understanding of the concepts.
